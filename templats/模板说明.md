## 设备实例化约定

模板文件只定义属性、服务和告警的语义。实际设备的 `deviceId`、`propertyId`、聚合来源与枚举实际值映射属于设备实例配置；平台在创建设备时保存模板快照和实际绑定到 SQLite 数据库。因此同一个模板可安全实例化为多台设备，彼此的点位映射不会覆盖。

设备实例中的属性绑定使用对象结构：

```json
{
    "method": "EPT",
    "sources": [
        { "deviceId": "raw-pcs-a01", "propertyId": "voltage" }
    ]
}
```

`enum` 属性只支持 `EPT` 直接绑定；数值属性可使用 `SUM`、`AVG`、`MIN`、`MAX`、`AND`、`OR`、`NOT` 等聚合方法。服务绑定为单一对象，告警绑定为来源数组。
