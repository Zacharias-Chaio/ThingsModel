# EMS物模型技术说明
## 1.物模型概念
### 1. 物模型(thing model)
    对一个设备、站点的数字化描述。包括以下内容：物的档案信息、物的状态信息、属性、服务、事件、组件。

### 3. 功能技术要求
#### 3.1 物的档案信息(Profile)
    - 定义
        设备、站点身份详情的静态描述，包含设备及站点身份标识和设备描述信息。
    - 组成元素
        1. 名称:（设备、或者站点名称）
        2. 描述: 
        3. 接口类型：北向接口 enum
        4. 协议类型：北向协议 enum
#### 3.2 物的状态信息（Status Information）
    - 定义
        设备在线/离线、激活/未激活等状态，状态信息的定义与属性定义的格式一致， 用于描述设备
    - 组成以元素
        1. 在线状态
        2. 激活状态
#### 3.3 属性（Property）
    - 定义
        描述设备静态或动态数据。 
    - 组成元素
        1. 属性ID: 一般属性的英文名称， 不可以重复 string
        2. 名称：string
        3. 描述
        4. 数据类型：int8/16/32/64， uint8/16/32/64, float,double, enum, bitmap, array
        5. 单位
        6. 基数： 默认0， 
        7. 系数：
        8. 精度
        9. 读写标志
#### 3.4 行为 Action  
    - 定义
        代表设备可对外提供的能力，用于描述设备可执行的动作或任务。
        默认提供设备在线状态，设备激活状态的获取接口方法， 自动生成属性为写标志的方法，用于控制
    - 组成元素  
#### 3.5 事件 Event
    - 定义
        设备运行时产生的各类事件信息的数据，包含告警、故障、信息等类型。
        事件是一类告警规则，与实际属性进行绑定
    - 组成元素
        1. 事件ID: 
        2. 事件名称：
        3. 事件描述
        4. 事件等级：提示，告警，故障
        5. 事件类型：等值，越上限，越下限
#### 3.6 组件 Componet
    -定义
        是基于物的自定义属性，自定义行为，自定义事件的一个集合。
