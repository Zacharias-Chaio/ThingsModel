模型介绍
了解物模型的基本概念与结构，帮助您更好地完成配置向导
物模型（Thing Model） 物模型是物理世界中实体（如传感器、设备等）在数字世界（云端）的标准化、结构化的抽象描述。 它本质上是一套通用的“数据语言”，让不同厂商、不同类型的物理实体能用同一种“语言”与软件应用对话。 在能源物联网领域中，EMS聚合多个设备的物模型形成站点级物模型，帮助实现跨设备、跨协议的数据统一接入与管理。
档案信息 Profile

关于物联网中“物的档案信息”，它通常指一个设备或“物”的静态、基础性的身份描述信息。 物的档案信息主要回答了“这个设备是谁”这一根本性问题

指令 Command

指令用于描述“物”与南向系统之间的数据交互规则，当前以列表化方式管理，并按协议规则生成请求报文与预期响应摘要。

属性 Property

描述“物”的静态或动态数据，用于表示设备的状态、测量值或其他可观测特性。“物”的属性主要回答了“现在怎么样？”的问题

行为 Action

“行为”是一个概括性术语，在不同平台或语境下，也会被称为“服务 (Service)”或“命令 (Command)”。它是对“物”可被外部调用的能力或方法的数字化定义，允许应用或用户向“物”下发指令来执行某个特定的操作。

事件 Event

在物模型里，“事件”是设备主动上报信息的核心机制。“设备发生了什么值得我注意的事情？” 它是设备主动、自发向平台或应用发出的通知，通常带有明确的触发条件和时间戳。

配置流程
  1. 填写设备档案信息（名称、协议等）
  2. 配置采集指令
  3. 定义属性数据点
  4. 启用 / 禁用行为方法
  5. 配置事件告警规则
  6. 预览并导出 JSON / YAML 文件
